# WebUI 与 WPF 功能对齐缺失清单（2026-02-26）

## 1) 审计范围

- WPF 入口：`MainWindow.xaml` 导航菜单
- WebUI 入口：`Service/Remote/WebRemoteIndex.html`
- Web API：`Service/Remote/WebRemoteService.cs`

## 2) 总览结论

- 当前 WebUI 主要覆盖：
  - 运行控制（启动/停止/暂停/继续/取消）
  - 基础开关（自动拾取/自动剧情/自动钓鱼/自动烹饪/自动吃药/快捷传送/地图遮罩）
  - 部分独立任务（七圣、伐木、自动战斗、自动秘境、幽境危战、全自动钓鱼、音游、圣遗物分解、网格图标、兑换码）
  - 调度器（配置组运行、增删改、添加脚本、组内项目编辑）
  - 一条龙（配置选择、执行、保存、克隆、基础字段）
- 仍有大量 WPF 页面级功能未进入 WebUI，且已有覆盖模块中也存在“高级配置缺失”。
- WebUI 当前未实现分页（长列表一次性渲染）。

## 3) 整页缺失（Web 无对应页面）

以下功能在 WPF 有独立入口，但 WebUI 未提供对应页面或导航入口：

1. AI 问答页（AI 聊天、设置、清空记录）
2. 启动页完整能力（截图模式、触发间隔、推理设备、窗口选择、联动启动参数等）
3. JS 脚本管理页（独立列表页）
4. 地图追踪管理页（树形任务管理、开发者工具入口）
5. 键鼠录制回放页
6. 辅助操控页（一键宏、转圈圈、快速强化等）
7. 快捷键配置页
8. 通知配置页（Webhook/WS/飞书/Telegram/邮箱/钉钉等）
9. 通用设置页（语言、遮罩行为、MCP 全配置、脚本安全策略等）

## 4) 部分覆盖但明显缺失

### 4.1 实时触发（TriggerSettings）仅保留了“开关”，缺少高级项

Web 仅支持基础布尔开关；WPF 的以下能力未暴露到 Web：

- 自动拾取：极速模式、OCR 引擎、黑白名单编辑、自定义拾取键
- 自动剧情：后台模式、画中画、优先选项文本、多项延时、自动提交/自动关闭弹窗等细项
- 自动邀约：分支与延时配置
- 自动吃药：触发间隔/吃药间隔
- 快速传送：点击间隔、等待传送面板延时、快捷键传送开关
- 冷却提示：角色 CD 规则、样式坐标/颜色/大小参数

### 4.2 独立任务（TaskSettings）仅覆盖主参数，缺高级配置与个别任务

- 未覆盖任务：自动地脉花（完整缺失）
- 自动战斗：结束检测、快速检测、色彩阈值、盾奶位、掉落扫描、超时等大量高级参数缺失
- 自动秘境：战后延时、寻树模式、小步伐行走、复活重试等缺失
- 幽境危战：与秘境类似的高阶树脂与流程细项未完整暴露
- 钓鱼：鱼种筛选等细节缺失

### 4.3 调度器（ScriptControl）有基础 CRUD，但缺进阶能力

- 缺少：复制组、从此处连续执行、日志分析、根据文件夹更新、任务倒序、导出控制文件变更
- 缺少：Shell 类型任务在 Web 侧的完整编辑流程
- 缺少：项目级“修改通用配置/修改 JS 自定义配置”等

### 4.4 一条龙（OneDragon）缺地脉花与任务编排能力

- Web 有基础字段编辑，但缺：
  - 一条龙中的地脉花运行日历与类型/国家配置
  - 任务列表的增删改排序能力（WPF 支持任务列表管理）
  - 删除配置（Web 只有刷新/执行/保存/克隆）

## 5) API 层缺口（导致 Web 无法补齐）

`WebRemoteService` 当前公开 API 只覆盖有限范围（基础开关、部分任务设置、调度器基础、一条龙基础、日志/画面）。

明显缺失：

- 无 `auto_leyline` 任务路由与设置路由
- 无 Trigger 高级配置对应路由
- 无 Notification / CommonSettings 大量配置路由
- 无 Hotkey / Macro / KeyMouseRecord / JsList / MapPathing 管理路由

另外存在“已实现处理函数但未挂路由”的情况：

- `HandleConfigGetAsync`
- `HandleConfigSetAsync`
- `HandleScriptGroupGetAsync`
- `HandleScriptGroupSaveAsync`

这会影响后续以“通用配置路径 API”快速补齐前端配置能力。

## 6) 分页现状

WebUI 目前没有分页机制，以下区域数据量大时会一次性渲染：

- 脚本组列表 `#scriptGroups`
- 组详情 `#groupDetail`
- 脚本库列表 `#libraryJs` / `#libraryKeyMouse` / `#libraryPathing`
- 日志区仅做行截断，不是分页

## 7) 建议落地顺序（务实）

1. 先补 API 能力：
   - 打通通用配置读写路由（受 allowlist 约束）
   - 补齐 `auto_leyline` 任务与设置 API
2. 再做 WebUI 结构升级：
   - 增加左侧导航页签，按 WPF 页面分组
   - 引入分页组件（至少用于调度器与脚本库）
3. 再补缺失页面：
   - 优先 Trigger 高级项、Task 高级项、一条龙地脉花
   - 再补 JS/MapPathing/KeyMouseRecord
4. 最后处理高风险页面：
   - Notification / CommonSettings / Hotkey / Macro（需权限与安全策略）
